시뮬레이터 UI/UX 개선 — 차트 뷰 모드, 상품 분석 구성품 표시

## 완료된 작업

### 1. 차트 뷰 모드 토글 (달성률 % ↔ 개수)
- **SystemChart**: `viewMode` 상태 추가 (`"rate"` / `"count"`)
- 등급 토글 버튼 옆에 pill 스타일 "달성률 %" / "개수" 전환 버튼 추가
- **달성률 모드**: 기존과 동일 (Y축 %, 툴팁 `90% (14.4 / 16)`)
- **개수 모드**: Y축 실제 보유 개수, 차트가 `_raw` 값 사용, 툴팁 `14.4 / 16`
- Y축 자동 스케일: 모드에 맞게 범위 조정
- PLC 100% 참조선은 달성률 모드에서만 표시
- **SystemSummaryTable**: `viewMode` prop 전달받아 개수/달성률 각각 다르게 표시
- CSS: `.chart-controls`, `.view-mode-toggles`, `.view-mode-btn` 스타일 추가

### 2. 상품 분석 탭 — 패키지 구성품 아코디언
- 패키지 행 클릭 시 아래에 **전체 구성품** 펼침/접힘 (아코디언)
- 행 왼쪽 `▶`/`▼` 화살표로 열림 상태 표시
- 구성품은 연한 배경 + 들여쓰기로 시각 구분

### 3. 전체 구성품 표시 (소환권 외 아이템 포함)
- **문제**: 기존에는 소환권만 필터링해서 `items`에 저장 → 젬, 딸기 파르페 등 미표시
- **해결**: `rawItems` 필드 추가 (Excel 원본 아이템명 + 수량 전부 포함)
  - `excelParser.js`: 브라우저 파싱 시 모든 아이템을 `rawItems`에 저장
  - `gen_packages3.py`: 정적 packages.js 재생성 시에도 `rawItems` 포함
  - `packages.js`: 46개 패키지 모두 `rawItems` 필드 추가
  - `App.jsx`: 구성품 표시 시 `rawItems` 우선 사용, 없으면 `items` 폴백
